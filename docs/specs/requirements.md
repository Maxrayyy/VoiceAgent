# 飞机维修领域 RAG 语音问答 Agent — 需求文档

## 1. 项目概述

构建一个面向飞机维修领域的语音问答智能助手（Agent），维修技术人员可通过语音提问，系统实时识别语音、结合领域知识库检索增强生成（RAG）回答问题，并将回答以语音形式播报。整个链路实现流式处理，最大化降低端到端延迟。

## 2. 目标用户

- 飞机维修工程师 / 技术人员
- 维修培训学员
- 质量检查人员

## 3. 核心使用场景

| 场景 | 描述 |
|------|------|
| 维修手册查询 | 技术人员在维修现场，双手被占用，通过语音询问特定机型的维修步骤、拆装顺序、力矩参数等 |
| 故障排查辅助 | 描述故障现象，系统从知识库中检索相关故障树/排故流程，给出排查建议 |
| 规章标准查询 | 查询适航指令（AD）、服务通告（SB）、维修方案等技术文档中的具体条款 |
| 工具/耗材查询 | 查询特定维修任务所需的工具清单、耗材规格、件号等 |

## 4. 功能需求

### 4.1 语音输入（STT）

- **FR-01**: 支持实时流式语音识别，用户说话过程中即开始转写
- **FR-02**: 支持中文普通话识别，兼容维修领域专业术语
- **FR-03**: 支持端点检测（VAD），自动判断用户说完
- **FR-04**: 支持中间结果展示，用户可看到实时转写文本
- **FR-05**: 音频格式支持 PCM 16kHz 16bit 单声道

### 4.2 知识检索（RAG）

- **FR-06**: 支持导入飞机维修领域文档（PDF、Word、TXT 等）建立向量知识库
- **FR-07**: 用户问题经向量化后在知识库中检索相关文档片段
- **FR-08**: 检索结果作为上下文注入 LLM Prompt，增强回答的准确性和专业性
- **FR-09**: RAG 优化（如 rerank、混合检索、chunk 策略等）后期迭代

### 4.3 大模型推理（LLM）

- **FR-10**: 使用阿里云 Qwen 模型（通过 OpenAI 兼容接口调用）
- **FR-11**: 支持流式输出（stream=True），逐 token 返回
- **FR-12**: 系统提示词针对飞机维修领域定制，确保回答专业、准确、安全
- **FR-13**: 支持多轮对话，保持上下文连贯
- **FR-14**: LLM 流式输出的文本片段直接流式传入 TTS

### 4.4 语音输出（TTS）

- **FR-15**: 使用阿里云 CosyVoice 模型进行流式语音合成
- **FR-16**: 支持双向流式合成，边接收 LLM 文本边合成音频
- **FR-17**: 合成音频实时播放，无需等待全部合成完毕
- **FR-18**: 音色清晰自然，适合技术内容播报

### 4.5 用户交互

- **FR-19**: 提供 Web 界面，包含录音按钮 / 实时转写显示 / 回答文本显示 / 音频播放
- **FR-20**: 支持打断功能——用户在系统播报时可重新开始提问
- **FR-21**: 显示 RAG 检索到的参考文档来源

## 5. 非功能需求

| 编号 | 需求 | 指标 |
|------|------|------|
| NFR-01 | 端到端延迟 | 用户说完到开始听到回答 < 3秒（不含 RAG 检索优化前） |
| NFR-02 | STT 识别准确率 | 通用场景 > 95%，专业术语通过热词优化提升 |
| NFR-03 | 并发支持 | 初期支持单用户，后续可扩展 |
| NFR-04 | 可用性 | 系统 7x24 可用，依赖阿里云服务 SLA |
| NFR-05 | 安全性 | API Key 不暴露在前端，所有密钥通过环境变量管理 |

## 6. 技术约束

- STT: 阿里云智能语音交互（NLS SDK，WebSocket 协议）
- TTS: 阿里云 CosyVoice（DashScope SDK，WebSocket 协议）
- LLM: 阿里云 Qwen 模型（OpenAI 兼容接口，HTTP SSE 流式）
- RAG: 本地向量数据库 + Embedding 模型（初期简单实现）
- 后端: Python（FastAPI / WebSocket）
- 前端: Web（HTML/JS，WebSocket 通信）

## 7. 数据流概览

```
用户麦克风 → [音频流] → STT(流式识别) → [文本] → RAG(向量检索) → [上下文+问题]
    → LLM(流式生成) → [文本流] → TTS(流式合成) → [音频流] → 用户扬声器
```

## 8. 里程碑规划

| 阶段 | 内容 | 备注 |
|------|------|------|
| P0 | STT + LLM + TTS 流式链路打通 | 核心链路，无 RAG |
| P1 | 基础 RAG 接入（简单向量检索） | 导入维修文档，基础检索 |
| P2 | Web 前端界面 | 完整用户交互界面 |
| P3 | RAG 优化 | rerank、chunk 优化、混合检索等 |
| P4 | 生产化 | 部署、监控、日志、安全加固 |
