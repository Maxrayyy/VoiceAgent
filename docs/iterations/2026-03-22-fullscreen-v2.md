# 全屏语音界面改造（版本 2）

## 目标
- 打造单一连续画布：去除所有可见容器、分割线，保持浅米白背景与极简工业风。
- 顶部居中波形横跨主要宽度，实时响应麦克风输入与 TTS 播放，始终可见。
- 左上角“飞机维修助手”与功能入口合成一个可折叠控件，控制左侧滑出面板。
- 侧栏从左侧滑入，仅承载辅助功能（打断、清除、参考来源等），并与品牌文字同步收起/展开。
- 对话呈现：
  - 初始完全留白；
  - 用户识别文本右对齐、AI 回复左对齐，均为无边框轻量文本；
  - 无对话时自动隐藏所有文本；
- 页面加载即自动开始录音，所有交互通过语音或悬浮控件完成。

## 代码变更范围
1. `src/server/static/index.html`
   - 新的顶层结构：品牌+入口组件、顶部波形、对话画布、隐藏控件层。
   - 左侧侧栏元素重建（滑层 + 控件 + 来源列表），与品牌入口联动。
2. `src/server/static/style.css`
   - 统一背景和排版，定义“单一画布”感。
   - 波形、对话文本、左侧入口、侧栏滑动等样式。
   - 用户/AI 文本左右对齐、自动淡入，空时隐藏。
3. `src/server/static/app.js`
   - 自动录音 + 未授权提示。
   - 波形既监听麦克风也监听 TTS：
     - STT 时采集媒体流；
     - TTS 播放时从播放 AudioContext 中读取能量并驱动波形。
   - 左侧面板折叠逻辑与入口联动。
   - 重新实现对话堆栈，控制用户/AI文本左右排列及空状态。
   - 控件按钮放置在侧栏内，维持 interrupt / clear / reference 功能。

## 验证清单
- 打开页面即弹出麦克风授权；允许后波形跳动，无多余 UI。
- 左上角品牌与入口紧邻，点击或悬停入口时左侧面板平滑展开，内容无边框。
- 说话后文本分别贴靠左右出现，停止说话后数秒未说话再次自动回归留白（若无文本）。
- 播放 TTS 时，即便用户未说话，波形仍随 AI 音频跳动。
- 清除上下文后，对话文本全部消失。
